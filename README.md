# Сайт прогнозов на матчи ПО КХЛ

Мини-приложение на Flask для компании друзей:
- личный кабинет пользователя (профиль, статистика, место в таблице);
- отдельная страница прогнозов по сериям;
- отдельная страница результатов и лидерборда;
- отдельная LIVE-страница с автообновлением, детектором изменений счёта и подготовкой текста для Telegram-уведомлений;
- отдельная страница дерева ПО с сериями и счетами;
- админ-функции для управления сериями, дедлайнами прогнозов, результатами и пользователями.

## Ключевая логика по сериям
- Серия играется до 4 побед (максимум 7 матчей).
- Пользователь задаёт счёт серии и выбирает победителя в каждом матче серии (система показывает только нужное число матчей).
- Очки считаются по серии: победитель серии, точный счёт серии и попадания по отдельным матчам.
- Для серии задается только **дедлайн прогноза (МСК)**, отдельные даты матчей для прогноза не обязательны.

## Роли
- **Обычный пользователь**: профиль + прогнозы + просмотр результатов/дерева.
- **Админ**: те же возможности, плюс:
  - `/admin/matches` — создать/удалить серию и задать дедлайн прогноза;
  - `/admin/results` — проставлять финальные счета матчей;
  - `/admin/users` — блокировать, разблокировать, удалять простых пользователей.

## Валидация имен
- Логин и отображаемое имя: только латиница/цифры/`_`, длина 3–24.
- Отображаемое имя уникально и используется в общей таблице результатов.

## Быстрый запуск (локально)
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

## Быстрый деплой на VPS через Docker
```bash
git clone https://github.com/R0uxgar0ux/PO_KHL.git
cd PO_KHL
docker compose up -d --build
```

Сайт будет доступен по адресу `http://<VPS_IP>:5000`.

### Если `docker compose build` падает на `pip install` (Temporary failure in name resolution)
- В `docker-compose.yml` уже добавлены:
  - `build.network: host` (берёт DNS хоста при сборке);
  - `dns: [1.1.1.1, 8.8.8.8]` для контейнера;
  - `PIP_INDEX_URL=https://pypi.org/simple`.
- Если проблема сохраняется, проверьте DNS на самом сервере:
  ```bash
  cat /etc/resolv.conf
  getent hosts pypi.org
  ```
- Для Docker daemon можно явно прописать DNS в `/etc/docker/daemon.json` и перезапустить Docker:
  ```json
  {
    "dns": ["1.1.1.1", "8.8.8.8"]
  }
  ```
  ```bash
  sudo systemctl restart docker
  ```

## LIVE-голы и подготовка к Telegram-боту
- Переменная `RESULTS_FEED_URL` включает LIVE-фид (JSON в формате `series -> matches`).
- Страница `/live`:
  - показывает активные матчи и последние завершённые за 7 дней;
  - автоматически обновляется (по умолчанию каждые 20 секунд);
  - подсвечивает новые голы между обновлениями;
  - показывает текстовые заготовки сообщений для будущего Telegram-бота.
- JSON API для интеграции бота: `GET /api/live` (требуется авторизация в веб-приложении).

## Логотипы команд
- В репозитории уже лежат локальные логотипы в `static/team_logos/` (включая `default.svg`), поэтому Docker-сборка не зависит от внешних URL.
- Если нужно обновить PNG/GIF логотипы из сети вручную, можно выполнить: `./scripts/fetch_team_logos.sh ./static/team_logos`.


## TypeScript модуль подсчета очков
Добавлен отдельный модуль бизнес-логики (без привязки к UI):
- `ts/src/predictorScoring.ts` — типы, валидация, подсчет очков, турнирная таблица.
- `ts/tests/scoring.test.js` — тестовые сценарии.
- `ts/examples/exampleData.ts` — пример входных данных и результата подсчета.

Команды:
```bash
npm run test:ts
npm run example:ts
```
